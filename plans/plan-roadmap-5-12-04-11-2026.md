#### Cycle 1.1 — Fix agent memory save (Item 7)

The bug: `return response;` is emitted in bodyCode BEFORE the postamble that saves conversation history. The postamble is dead code.

🔴 **Test first** (clear.test.js):
```javascript
describe('Agent memory save order', () => {
  it('conversation save comes before return', () => {
    const src = `build for javascript backend

agent 'Helper' receives question:
  remember conversation context
  response = ask claude 'Help' with question
  send back response`;
    const r = compileProgram(src);
    const js = r.serverJS || r.javascript;
    // Find the return statement and the history push
    const returnIdx = js.indexOf('return response');
    const saveIdx = js.indexOf('_history.push');
    expect(saveIdx).not.toBe(-1);
    expect(returnIdx).not.toBe(-1);
    expect(saveIdx < returnIdx).toBe(true);
  });
});
```

🟢 **Fix:** In `compileAgent()` in compiler.js, the postamble code (conversation save) is appended after bodyCode. But bodyCode contains `return response;`. Fix by:
- Finding the last `return` in bodyCode
- Inserting the postamble BEFORE that return
- Or: remove `return` from bodyCode and add it at the end of postamble

Implementation: In the section that builds the final function string (~line 2359-2361), replace the bodyCode's last `return response;` with the postamble + return:
```javascript
// Before the final return assembly, inject postamble before the last return

  const lastReturn = bodyCode.lastIndexOf('return response');
  bodyCode = bodyCode.slice(0, lastReturn) + postamble + '\n' + bodyCode.slice(lastReturn);
  postamble = ''; // already injected
}
```

## 🚨 EDGE CASES

| Scenario | How we handle it |
|----------|-----------------|
| Agent with memory but no `send back` / no return | Postamble still saves — just no return to move |
| Tool with 0 parameters | Empty properties `{}`, empty required `[]` |
| Search with empty query string | Return all records (empty filter matches everything) |
| `has many` without corresponding `belongs to` | Works independently — just generates endpoint that filters by parent field |
| Python app with no pages | No static serving code emitted |
| Guardrail patterns with regex special chars | Escape them with `\` in the compiled regex |
| `broadcast` outside WebSocket context | Compile but will fail at runtime (wss not defined) — add validator warning |
| String concat with numbers | JS handles `+` polymorphism natively |
