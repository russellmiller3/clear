A Client Invoice System in 30 Lines

This is a real, working API. Auth, validation, CRUD, data persistence. Compiles to Express.js.

build for javascript backend

create an Invoices table:
  client is text, required
  amount (number), required
  status, default "draft"
  due_date
  created_at_date, auto

when user calls POST /api/invoices sending invoice:
  requires auth
  validate invoice:
    client is text, required
    amount is number, required
  set saved to save invoice as new Invoice
  send back saved with success message

when user calls GET /api/invoices:
  requires auth
  set all_invoices to get all Invoices
  send back all_invoices

when user calls PUT /api/invoices/:id sending updates:
  requires auth
  set updated to update Invoice with this id using updates
  send back updated

Change "Invoices" to "Bookings", adjust the fields, deploy. New client, same quality.

When bugs happen, the AI fixes them

AI-generated code has 1.7x more bugs than human-written code (OWASP 2026). 86% of AI code fails basic XSS defense. 60% of bugs are invisible -- the code runs but produces wrong output.

Clear solves this with a runtime error translator. When compiled code throws an error, the app tells the AI exactly what went wrong, which Clear file and line caused it, and how to fix it.

Three levels of context

Level 1 (production): The user sees "Something went wrong." No database internals leaked.

Level 2 (development): The AI sees the Clear source line, a plain-English hint, and a suggested fix -- a minimal diff that solves the problem.

Level 3 (deep debug): Full schema context, sanitized request data, cross-file dependency map. PII is auto-redacted.

The AI reads the error. The AI writes the fix. You review it.

The error translator gives AI agents the same context a senior developer would have: what tables exist, what fields are required, which file defined the endpoint, and what the frontend is sending. No more guessing.

Install Clear

npm install -g clear-lang

Build from a template

clear init my-app --template invoice-engine
cd my-app
clear build main.clear
node server.js

Or start from scratch

clear init my-app
cd my-app
# edit main.clear
clear build main.clear
node server.js

Deploy

clear package main.clear --out deploy/
# push to Vercel, Railway, or any Node.js host
