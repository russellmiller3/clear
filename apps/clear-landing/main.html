A Client Invoice System in 30 Lines

This is a real, working API. Auth, validation, CRUD, data persistence. Compiles to Express.js.

build for javascript backend

create an Invoices table:
  client is text, required
  amount (number), required
  status, default "draft"
  due_date
  created_at_date, auto

when user calls POST /api/invoices sending invoice:
  requires auth
  validate invoice:
    client is text, required
    amount is number, required
  set saved to save invoice as new Invoice
  send back saved with success message

when user calls GET /api/invoices:
  requires auth
  set all_invoices to get all Invoices
  send back all_invoices

when user calls PUT /api/invoices/:id sending updates:
  requires auth
  set updated to update Invoice with this id using updates
  send back updated

Change "Invoices" to "Bookings", adjust the fields, deploy. New client, same quality.

AI Agents Built In

Clear apps can call AI models mid-flow. Score leads, classify support tickets, moderate content, summarize documents -- all from readable code.

agent 'Lead Scorer' receiving lead:
  check lead's company is not missing, otherwise error 'Company is required'
  set lead's score to ask ai 'Rate 1-10 for enterprise potential.' with lead's company
  send back lead

when user calls POST /api/score sending lead_data:
  validate lead_data:
    company is text, required
    email is text, required
  set scored to call 'Lead Scorer' with lead_data
  set saved to save scored as new Lead
  send back saved with success message

The prompt is visible in the source. Your client can read it. You can audit it. No black-box AI.

Chain multiple agents into pipelines: screen, then score, then summarize, then save.

Install Clear

npm install -g clear-lang

Build from a template

clear init my-app --template invoice-engine
cd my-app
clear build main.clear
node server.js

Or start from scratch

clear init my-app
cd my-app
# edit main.clear
clear build main.clear
node server.js

Deploy

clear package main.clear --out deploy/
# push to Vercel, Railway, or any Node.js host
