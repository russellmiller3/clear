What Marcus is afraid of

Four fears.
Four answers built into the compiler.

Every "live editing" tool claims this works. The question is what actually prevents disaster. Here's what Clear does, mechanically, for each fear.

Fear 1

"What if I break the app?"

Every proposed change is classified before it runs. You see what category it falls in and exactly what it'll touch — before anything ships.

Additive
Adds a field/page/endpoint — nothing can break. Ships instantly.
Reversible
Removes a field, renames, relabels. Data stays. One-click un-hide.
Destructive
Permanently deletes data. Explicit opt-in only. Snapshot + "type DELETE" to confirm.
Fear 2

"What if the AI deletes other code?"

Meph can't do what Bolt does — rewrite a whole file and accidentally delete your approval logic. Meph's edit tool is a patch, not a rewrite.

main.clear (changed lines only)
...
+ a region from 'NA', 'EMEA', 'APAC'
+ default region is 'NA'
...
Everything Meph did touches only these lines. Your approval routing, email rules, and agent instructions are physically untouched — the compiler enforces it.
Fear 3

"What about Jenna's half-filled form?"

When the app reloads with the new version, the compiler preserves every value the user already typed. New fields appear empty. Existing input survives.

The promise

Never lose a user's form data
when you change the app.

Clear's compiler is additive-by-default. When you rename or change a column, it doesn't drop the old one — it adds the new one alongside, copies the data, and keeps both readable until every consumer has moved over. Then the old one retires.
