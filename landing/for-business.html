Why you can trust what I ship

Most custom software is a black box. This isn't.

Source code you can actually read
# Expense approval workflow
create a Expense table:
  description, required
  amount (number), required
  submitter, required
  approved, default false

when user sends expense to /api/expenses:
  requires login
  validate expense:
    description, required
    amount, between 0 and 100000
  save expense as new Expense
  email manager: 'New expense to review'

test:
  can user submit an expense
  submitting without amount should be rejected
  creating an expense should require login
Not JavaScript. Not Python. English.

Your IT team can read the source file and know exactly what the app does. No reverse-engineering. No tribal knowledge.

Security bugs are compile errors

SQL injection, auth bypass, mass assignment, PII in error logs — the compiler refuses to build if any are present. Not "best effort." Mathematically impossible to ship.

Tests prove it works

Every endpoint, every button, every agent guardrail gets an auto-generated test with an English name. I don't mark a feature "done" until the tests are green.

You own it forever

Source in your git. Runs on your cloud. If we part ways, the app keeps running. Any competent engineer can maintain it — the source is English.

Common questions

What happens if you're not available or something happens to you?

The source is in your git repo in plain English. Any competent developer (or AI assistant) can read and modify it. You're not buying access to me — you're buying a working system you understand and own.

Do you do on-prem deployments for compliance reasons?

Yes. The compiled output is standard Node.js or Python. It runs anywhere — your AWS VPC, Azure tenant, on-prem kubernetes, wherever your InfoSec team approves. No external call-home.

Can I see an example before committing?

Absolutely. On the discovery call I'll walk you through a live build — compile, tests, deploy — in under 10 minutes. You see exactly what the source looks like and what you'd own at the end.

What about AI safety / hallucinations in the agent tools?

Agents have compiler-enforced guardrails: allowed tools, blocked arguments, role restrictions, validated inputs. Prompt injection attempts are logged. Every decision is auditable. This is not a wrapper around ChatGPT.

Who owns the intellectual property?

You do — the source code I write for you is yours. I keep the right to use the Clear language and compiler (which are my own work) for other projects. Straightforward work-for-hire.
