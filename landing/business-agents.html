Agents that can't go rogue

Three layers of protection. Compile-time, runtime, and audit trail.

Layer 1: Compile-time
Before the agent exists
  • ■ Tool validation — can use: delete_order but no function named delete_order exists? Compile error.
  • ■ Policy enforcement — must not: delete Orders checked against every tool the agent can call. Contradiction = compile error.
  • ■ Undefined agent calls — calling an agent that doesn't exist = compile error.
Layer 2: Runtime
While the agent runs
  • ■ Input filtering — block arguments matching 'drop|truncate' = regex guard on every tool input. Prompt injection blocked.
  • ■ Tool restrictions — agent can only call tools listed in can use:. No surprise API calls.
  • ■ Human approval — ask user to confirm pauses the agent and waits for human yes/no.
Layer 3: Observability
After the agent acts
  • ■ Decision logging — track agent decisions logs every AI call with timing, tokens, tool choices.
  • ■ Conversation memory — DB-backed history. See every conversation the agent had.
  • ■ Source maps — runtime errors map back to Clear line numbers. Debug in 40 lines, not 2,000.
