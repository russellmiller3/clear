Thesis

AI will write all software.
Safety should live in the compiler, not the model.

The problem with AI-generated code

LLMs can already write software faster than humans. Lovable, Bolt, Cursor, Devin — all generate working code from prompts. The hard problem isn't speed. The hard problem is trust.

How do you know the AI didn't ship a security hole? How do you know the agent won't delete your database? How do you know the output is the same as yesterday's?

Today's answer is hope. Hope the AI wrote secure code. Hope it didn't break something. Hope the output is right.

The compiler builds the cage

Our approach to alignment: don't try to make the model safe. Build a cage around its output. The AI writes whatever it wants in Clear. The compiler decides what ships.

The cage has 27 walls. Auth bypass? Won't compile. SQL injection? Impossible — queries are parameterized structurally. Agent trying to delete data? Blocked by guardrails the human wrote in English and the compiler enforces in code.

Inside the cage, the AI is free — creative, fast, capable. Outside the cage, nothing ships. As AI gets more capable, we add more walls. The cage and the capability co-evolve.

Architecture
Compiler

27 bug classes eliminated
at compile time.

Not warnings. Not lint rules. Compile errors. The program doesn't build until the bug is fixed. Fix a pattern once in the compiler, every app gets the fix on recompile.

Security — compile errors
✓SQL injection — all queries parameterized
✓Auth bypass — DELETE without login = won't compile
✓Mass assignment — unknown fields auto-stripped
✓CSRF — mutating endpoints without auth = error
✓PII in errors — passwords auto-redacted
✓XSS — all user input escaped in display
Correctness — caught before runtime
✓Undefined variables with typo suggestions
✓Type mismatches in math operations
✓Frontend-backend URL mismatches
✓Missing endpoint responses
✓Schema-frontend field mismatches
✓Agent tool reference validation
Business logic — common mistakes
✓Negative balance / stock warnings
✓Overbooking without capacity check
✓Deep property chain warnings
✓Expression complexity limits
✓Prompt injection blocking
✓Input validation auto-generated
Agent primitives

Agents as language constructs,
not library calls.

Tool use, memory, RAG, guardrails, pipelines, parallel execution, human approval gates, structured output, and multi-agent coordination — all as first-class syntax. The compiler validates agent references, tool bindings, and guardrail patterns at compile time.

Agent with tools + guardrails
agent 'Helpdesk' receives question:
has tools: lookup_product, create_ticket
knows about: Products
remember conversation context
block arguments matching 'delete|drop|truncate'
response = ask claude 'You are a customer
support agent. Help with products
and account issues.' with question
send back response
Multi-agent pipeline
pipeline 'Deal Review' with request:
step 'Enrich' with 'Enrichment Agent'
step 'Score' with 'Scoring Agent'
step 'Summarize' with 'Summary Agent'
agent 'Triage' runs every 1 hour:
pending = get all Requests where status is 'new'
for each req in pending:
result = call pipeline 'Deal Review' with req
send email to req's owner_email:
subject is 'New deal scored'
Guardrails
block arguments matching, must not, protect tables — compile-time constraints on agent behavior
Memory
remember conversation context, remember user preferences — DB-backed, per-user long-term memory
Human-in-loop
ask user to confirm — approval gates before irreversible actions. Policy, not code.
Coordination
Agents call agents. Pipelines chain steps. do these at the same time for parallel fan-out.
RL environment

A gym for AI that writes software.

Deterministic compiler. Structured errors as reward signal. Constrained action space. 20 curriculum tasks across 10 difficulty levels. The infrastructure for training AI systems that write correct, safe, auditable code.

What's built
Sandbox runner
Isolated child process with timeout and memory limits. Deterministic execution environment.
Curriculum
20 benchmark tasks (L1-L10). 63 tests. From "create a variable" to "build a multi-agent pipeline with auth."
Patch API
11 structured edit operations. Constrained action space — the AI edits programs through operations, not raw text. Add line, replace line, insert block, wrap in conditional.
Structured eval
compileProgram() returns JSON: error count, warning count, compiled output, test results, timing. The reward function.
Source maps
Runtime errors map back to Clear line numbers. The AI can read its own stack traces.
The loop
Blocker
No fine-tuning access. The gym is ready but can't train athletes in it yet. When frontier model APIs open training endpoints, Clear is the environment.
Trajectory

From writing code
to setting constraints.

1
Today — AI writes, compiler catches, human reviews

AI generates 100-line .clear files. Compiler catches security bugs and correctness errors. Human reviews the readable source. Ships.

2
Next — AI writes complex systems, evals grade, human reviews exceptions

AI generates 500-line multi-agent systems. Compiler validates. Evals grade quality against rubrics. Human reviews only what the evals flag.

3
Later — AI writes, compiles, tests, deploys, monitors autonomously

Human sets policy: block arguments matching, must not:, requires role. Compiler enforces it. AI handles everything else. The human moves from writing code to setting constraints.

4
Eventually — AI writes agents that write .clear files

Self-improving software factory, constrained by a deterministic compiler. The compiler is the alignment mechanism. As the compiler gets smarter, the AI gets safer. They co-evolve.

Intelligence in the compiler.
Safety encoded, not hoped for.
