Why retries collapse

Three properties the agent learns to trust.

01

Deterministic compile.

Same source → same output, every time. No "regenerate the React component" inference loops. The compiler is a function, not a model.

02

Errors are actionable.

Every compile error names the exact Clear line, what's wrong, and what canonical syntax to use. Agents fix on first read, not the third.

03

Source IS English.

No framework choices, no ceremony, no boilerplate. The model writes intent; the compiler handles the rest. Smaller surface, fewer mis-steps.

Install

Two minutes from zero to first compile.

1

Install the CLI.

Compiles .clear files, scaffolds projects, runs tests. Pure Node, zero dependencies.

npm install -g clear-cli Node 18+
2

Install the editor extension.

Syntax highlighting, autocomplete, inline diagnostics. Works in VSCode, Cursor, and any editor with an LSP client.

code --install-extension clearlang.clear-language VSCode + Cursor
3

Scaffold a project.

Opens with a working todo app — auth, database, frontend, tests, all from one .clear file.

clear init my-app && cd my-app && clear serve main.clear Opens at :3000
4

Run the browser walker. Compiler-derived. Zero hand-written tests.

Every clear build writes a Playwright script that drives every page, every nav click, every button, every table sort+filter, every detail-panel drilldown — and screenshots each route. No comparable internal-tool platform ships this.
